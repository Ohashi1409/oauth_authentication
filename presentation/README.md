# Apresentação sobre OAuth

Este diretório está reservado para armazenar a apresentação sobre OAuth Authentication.

## Como adicionar sua apresentação

1. Coloque seu arquivo PDF de apresentação neste diretório
2. Recomendamos nomear o arquivo de forma descritiva, por exemplo:
   - `apresentacao-oauth.pdf`
   - `oauth-authentication-presentation.pdf`
   - `oauth-slides.pdf`

## Formato suportado

- **PDF**: Formato recomendado para apresentações
- Outros formatos de apresentação (PowerPoint, Google Slides) também podem ser adicionados se necessário

## Estrutura sugerida da apresentação

Para complementar a documentação em markdown deste repositório, sua apresentação pode cobrir:

1. **Introdução ao OAuth**
   - O que é e por que usar
   - Problemas que resolve

2. **Conceitos Fundamentais**
   - Resource Owner, Client, Authorization Server, Resource Server
   - Access Tokens e Refresh Tokens

3. **Fluxos de Autorização**
   - Authorization Code Flow
   - Authorization Code Flow com PKCE
   - Client Credentials Flow
   - Comparação entre os fluxos

4. **Segurança**
   - Vulnerabilidades comuns
   - Melhores práticas
   - PKCE e sua importância

5. **Implementação Prática**
   - Exemplos de código
   - Demonstração ao vivo
   - Providers populares (Google, GitHub, etc.)

6. **OAuth vs OpenID Connect**
   - Diferenças entre autorização e autenticação
   - Quando usar cada um
